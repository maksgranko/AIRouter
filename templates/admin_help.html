Использование Роутера как OpenAI-Совместимого API

Данный роутер предоставляет эндпоинты, совместимые с OpenAI API. Это означает, что вы можете использовать существующие клиенты и библиотеки, предназначенные для работы с OpenAI, направив их на адрес этого роутера.

Основные Эндпоинты

  • /v1/chat/completions - для запросов к чат-моделям.
  • /v1/completions - для запросов к моделям дополнения текста (legacy).
  • /v1/embeddings - для получения векторных представлений текста.
  • /v1/models - для получения списка доступных моделей от активных модулей.
  • /v1/models/{model_id} - для получения информации о конкретной модели.
  • Другие эндпоинты, такие как /v1/moderations, /v1/images/generations, /v1/audio/transcriptions, /v1/audio/translations, также реализованы и будут работать, если соответствующий модуль их поддерживает.

Отправка Запросов

При отправке запроса, например, к /v1/chat/completions, важно указать параметр model в теле JSON-запроса. Этот параметр определяет, какой из подключенных модулей (и какая конкретная модель внутри этого модуля, если применимо) будет обрабатывать запрос.

  • Для использования модуля OpenAI: "model": "openai" или "model": "openai/gpt-3.5-turbo" (если модуль OpenAI зарегистрирован с именем "openai"). Роутер попытается найти модуль по полному имени или по имени сервиса.
  • Для использования модуля Gemini: "model": "gemini" или "model": "gemini/gemini-pro".
  • Если параметр model не указан, по умолчанию будет использован сервис openai.

Роутер не требует API ключа для доступа к своим эндпоинтам. Аутентификация происходит на уровне самих AI-сервисов (OpenAI, Gemini), ключи для которых управляются данным роутером.

Пример запроса с использованием curl


curl http://127.0.0.1:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai", 
    "messages": [
      {
        "role": "user",
        "content": "Привет, как дела?"
      }
    ]
  }'

Замените "openai" на "gemini" или другой идентификатор модели/сервиса, чтобы направить запрос соответствующему модулю.

Переменные Окружения

Эти переменные устанавливаются перед запуском приложения и определяют его базовое поведение. Изменения через UI для некоторых из этих настроек (прокси) действуют только до перезапуска.

ПеременнаяОписаниеЗначения по умолчанию
ADMIN_USERNAMEЛогин для доступа к панели администратора.admin
ADMIN_PASSWORDПароль для доступа к панели администратора.supersecret
USE_PROXIESВключает (true) или выключает (false) использование прокси-серверов.true
PROXY_ROTATION_MODEРежим ротации прокси: once (однократный обход списка) или cycle (циклический обход).once

Файлы Конфигурации

Эти файлы хранят списки ключей и прокси. Их можно редактировать напрямую или через панель администратора.

  • openai_keys.json: Список API ключей для OpenAI. Формат: JSON-массив строк.
    Пример: ["sk-xxxx", "sk-yyyy"]
  • gemini_keys.json: Список API ключей для Gemini. Формат: JSON-массив строк.
    Пример: ["ai-xxxx", "ai-yyyy"]
  • proxies.json: Список прокси-серверов. Формат: JSON-массив объектов.
    Каждый объект: {"type": "http|socks4|socks5", "url": "protocol://user:pass@host:port"}.
    Пример: [{"type": "http", "url": "http://proxy.example.com:8080"}, {"type": "socks5", "url": "socks5://user:pass@another.proxy:1080"}]

Управление через Панель Администратора

Настройки Прокси

  • Использование прокси: Позволяет временно (до перезапуска) включить или выключить использование всех прокси-серверов. Исходное значение берется из USE_PROXIES.
  • Режим ротации прокси: Позволяет временно (до перезапуска) изменить режим ротации. Исходное значение из PROXY_ROTATION_MODE.

Управление API Ключами

  • Отображает ключи для каждого сервиса (openai, gemini).
  • Позволяет добавлять новые ключи и удалять существующие.
  • Изменения немедленно сохраняются в соответствующие .json файлы и применяются в приложении.

Управление Прокси-серверами

  • Отображает список прокси из proxies.json.
  • Позволяет добавлять новые прокси (тип: http, https, socks4, socks5; URL).
  • Позволяет удалять существующие прокси.
  • Кнопка "Перезагрузить список прокси из файла" позволяет применить изменения, сделанные в proxies.json вручную, без перезапуска всего приложения.
  • Изменения (кроме перезагрузки) немедленно сохраняются в proxies.json и применяются.

Управление Модулями (Сервисами)

  • Позволяет включать или выключать каждый зарегистрированный модуль (сервис).
  • Отключенный модуль не будет доступен для API запросов (например, /v1/chat/completions не сможет использовать модели этого сервиса, а /v1/models не будет их показывать).
  • Изменения действуют до перезапуска приложения.

Логика Ротации

Приложение использует следующую логику при выполнении запросов к API:

  1. Берется текущий API ключ для запрошенного сервиса.
  2. Если прокси включены (USE_PROXIES=true и есть загруженные прокси):
    • Берется текущий прокси из списка.
    • Выполняется запрос.
      • При успехе - возвращается результат.
      • При ошибке ключа (аутентификация, права) - переход к шагу 3 (смена ключа).
      • При ошибке сети/соединения или лимита запросов - попытка сменить прокси (согласно PROXY_ROTATION_MODE). Если прокси закончились (в режиме once) или ротация выключена - переход к шагу 3 (смена ключа).
  3. Если прокси выключены:
    • Выполняется прямой запрос.
    • При успехе - возвращается результат.
    • При любой ошибке (ключ, лимит, сеть) - переход к шагу 3 (смена ключа).
  4. Смена API ключа: берется следующий ключ из списка для данного сервиса.
    • Если ключи закончились - возвращается ошибка клиенту.
    • Если новый ключ получен - список прокси сбрасывается на начало, и происходит возврат к шагу 2.
