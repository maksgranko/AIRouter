Использование Роутера как OpenAI-Совместимого API

Данный роутер предоставляет эндпоинты, совместимые с OpenAI API. Это означает, что вы можете использовать существующие клиенты и библиотеки, предназначенные для работы с OpenAI, направив их на адрес этого роутера.

Основные Эндпоинты

  • /v1/chat/completions - для запросов к чат-моделям.
  • /v1/completions - для запросов к моделям дополнения текста (legacy).
  • /v1/embeddings - для получения векторных представлений текста.
  • /v1/models - для получения списка доступных моделей от активных модулей.
  • /v1/models/{'{model_id}'} - для получения информации о конкретной модели.
  • Другие эндпоинты, такие как /v1/moderations, /v1/images/generations, /v1/audio/transcriptions, /v1/audio/translations, также реализованы и будут работать, если соответствующий модуль их поддерживает.

Отправка Запросов

При отправке запроса, например, к /v1/chat/completions, важно указать параметр model в теле JSON-запроса. Этот параметр определяет, какой из подключенных модулей (и какая конкретная модель внутри этого модуля, если применимо) будет обрабатывать запрос.

  • Для использования модуля OpenAI: "model": "openai" или "model": "openai/gpt-3.5-turbo".
  • Для использования модуля Gemini: "model": "gemini" или "model": "gemini/gemini-pro".
  • Если параметр model не указан, по умолчанию будет использован сервис openai.

Аутентификация для API роутера (/v1/...) управляется настройкой "Требовать API-ключ AIRouter" в админ-панели. Если включено, требуется Bearer токен. Ключи для самих AI-сервисов (OpenAI, Gemini) управляются роутером.

Пример запроса с использованием curl

curl http://127.0.0.1:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  # Если включено требование ключа AIRouter, добавьте:
  # -H "Authorization: Bearer ВАШ_AIROUTER_API_КЛЮЧ" \
  -d '{
    "model": "openai", 
    "messages": [
      {
        "role": "user",
        "content": "Привет, как дела?"
      }
    ]
  }'

Замените "openai" на "gemini" или другой идентификатор модели/сервиса.

Переменные Окружения (`.env` файл)

Эти переменные устанавливаются перед запуском приложения и определяют его базовое поведение.

ПеременнаяОписаниеЗначения по умолчанию
ADMIN_USERNAMEЛогин для доступа к панели администратора.admin
ADMIN_PASSWORDПароль для доступа к панели администратора.supersecret

Настройки прокси (USE_PROXIES, PROXY_ROTATION_MODE и др.) теперь в основном управляются через configs/settings.json и UI/API админ-панели.

Файлы Конфигурации (в папке configs/)

Эти файлы хранят списки ключей, прокси и другие настройки. Их можно редактировать напрямую или через панель администратора/API.

  • settings.json: Основной файл настроек (статусы прокси, модулей, требование ключа AIRouter).
  • openai_keys.json: API ключи для OpenAI. JSON-массив строк.
  • gemini_keys.json: API ключи для Gemini. JSON-массив строк.
  • airouter_api_keys.json: API-ключи для доступа к самому AIRouter. JSON-массив строк.
  • proxies.json: Список прокси. JSON-массив объектов ({"type": "...", "url": "..."}).

Управление через Панель Администратора и JSON API

Панель администратора (начиная с /admin/dashboard) и соответствующее JSON API (начиная с /api/admin/ui/) позволяют управлять всеми ключевыми аспектами роутера:

  • Настройками прокси (включение, режим ротации и т.д.).
  • API ключами для AI-сервисов и для самого AIRouter.
  • Списком прокси-серверов.
  • Статусами активности модулей AI-сервисов.
  • Требованием API-ключа для доступа к роутеру.

Подробное описание JSON API эндпоинтов доступно в файле `README.md`.
