# AI API Роутер

Этот проект представляет собой OpenAI-совместимый API роутер, который позволяет направлять запросы к различным AI-сервисам (модулям), таким как OpenAI и Gemini. Он поддерживает управление API ключами, использование прокси-серверов с ротацией, а также имеет веб-панель администратора для управления конфигурацией.

## Возможности

- **OpenAI-Совместимость**: Используйте существующие клиенты OpenAI для взаимодействия с этим роутером.
- **Модульная Архитектура**: Легко добавляйте поддержку новых AI-сервисов.
  - Предустановленные модули: OpenAI, Gemini.
- **Управление API Ключами**:
  - Хранение ключей в JSON-файлах (`configs/openai_keys.json`, `configs/gemini_keys.json`).
  - Автоматическая ротация ключей при ошибках.
  - Управление ключами через веб-панель.
- **Поддержка Прокси**:
  - Использование HTTP/HTTPS/SOCKS4/SOCKS5 прокси.
  - Список прокси в `configs/proxies.json`.
  - Ротация прокси при ошибках.
  - Настройка использования и режима ротации через переменные окружения или веб-панель.
- **Веб-Панель Администратора**:
  - Доступ по адресу `/admin/dashboard` (защищено HTTP Basic Authentication).
  - Просмотр и управление настройками прокси.
  - Просмотр, добавление и удаление API ключей.
  - Просмотр, добавление и удаление прокси-серверов.
  - Включение/выключение модулей AI-сервисов.
  - Страница справки (`/admin/help`).
- **Гибкая Конфигурация**:
  - Через переменные окружения (файл `.env`).
  - Через JSON-файлы в папке `configs`.
  - Автоматическое создание конфигурационных файлов по умолчанию.

## Установка

1.  **Клонируйте репозиторий (если это репозиторий):**

    ```bash
    # git clone [URL вашего репозитория]
    # cd [название папки проекта]
    ```

    Если у вас просто файлы, перейдите в папку с проектом.

2.  **Создайте и активируйте виртуальное окружение (рекомендуется):**

    ```bash
    python -m venv venv
    # Windows
    venv\Scripts\activate
    # macOS/Linux
    source venv/bin/activate
    ```

3.  **Установите зависимости:**
    Убедитесь, что у вас установлен Python 3.7+ и pip.
    ```bash
    pip install -r requirements.txt
    ```

## Начальная Настройка

1.  **Создайте файл `.env`:**
    Скопируйте файл `.env.example` в новый файл с именем `.env`:

    ```bash
    # Windows
    copy .env.example .env
    # macOS/Linux
    cp .env.example .env
    ```

    Откройте файл `.env` в текстовом редакторе и настройте переменные:

    - `ADMIN_USERNAME`: Логин для панели администратора (по умолчанию `admin`).
    - `ADMIN_PASSWORD`: **Обязательно измените** пароль для панели администратора на надежный.
    - `USE_PROXIES`: `true` или `false` для включения/выключения использования прокси (по умолчанию `true`).
    - `PROXY_ROTATION_MODE`: `once` или `cycle` для режима ротации прокси (по умолчанию `once`).

2.  **Конфигурационные файлы в папке `configs`:**
    При первом запуске приложения, если папка `configs` или файлы в ней отсутствуют, они будут созданы автоматически с содержимым по умолчанию (обычно пустые списки для ключей/прокси или дефолтные настройки).

    Вам нужно будет заполнить эти файлы вашими данными. Это можно сделать вручную, отредактировав файлы, или через панель администратора после первого запуска.

    - `configs/openai_keys.json`: Добавьте ваши OpenAI API ключи.
      Пример: `["sk-xxxxxxxxx", "sk-yyyyyyyyy"]`
    - `configs/gemini_keys.json`: Добавьте ваши Gemini API ключи.
      Пример: `["ai-xxxxxxxxx", "ai-yyyyyyyyy"]`
    - `configs/proxies.json`: Если вы используете прокси, добавьте их сюда.
      Пример: `[{"type": "http", "url": "http://user:pass@host:port"}, {"type": "socks5", "url": "socks5://host:port"}]`
    - `configs/settings.json`: Хранит настройки, измененные через UI (статусы прокси, режим ротации, статусы модулей). Обычно создается и управляется автоматически.

## Запуск Приложения

Используйте Uvicorn для запуска FastAPI приложения:

```bash
uvicorn main:app --reload
# или же
python -m uvicorn main:app --reload
```

- `main`: имя Python файла (main.py).
- `app`: объект FastAPI (`app = FastAPI()`) в вашем файле `main.py`.
- `--reload`: включает автоматическую перезагрузку сервера при изменениях в коде (удобно для разработки).

После запуска сервер будет доступен по адресу `http://127.0.0.1:8000` (или другому адресу/порту, если настроено иначе).

## Использование

### API Роутера

- **Адрес API**: `http://127.0.0.1:8000/v1/...`
- **Эндпоинты**:
  - `/v1/chat/completions` (POST)
  - `/v1/completions` (POST)
  - `/v1/embeddings` (POST)
  - `/v1/models` (GET)
  - `/v1/models/{model_id}` (GET)
  - и другие.
- **Выбор модуля/модели**: В теле JSON-запроса указывайте параметр `"model"`.
  - Пример для OpenAI: `{ "model": "openai/gpt-3.5-turbo", "messages": [...] }`
  - Пример для Gemini: `{ "model": "gemini/gemini-pro", "messages": [...] }`
  - Если указать только имя сервиса (например, `"model": "openai"`), будет использована модель по умолчанию для этого сервиса.
- **Аутентификация API**: Не требуется для доступа к роутеру. Роутер использует ключи из файлов `configs/*_keys.json` для аутентификации на стороне AI-сервисов.

### Панель Администратора

- **Адрес**: `http://127.0.0.1:8000/admin/dashboard`
- **Аутентификация**: Используйте логин и пароль, заданные в переменных окружения `ADMIN_USERNAME` и `ADMIN_PASSWORD`.
- **Возможности**:
  - Просмотр и изменение настроек использования прокси и режима ротации.
  - Просмотр, добавление и удаление API ключей для OpenAI и Gemini.
  - Просмотр, добавление и удаление прокси-серверов.
  - Включение и выключение модулей (сервисов OpenAI, Gemini).
- **Страница Справки**: `http://127.0.0.1:8000/admin/help` - содержит подробную информацию о конфигурации и использовании.

## Устранение Неполадок

- **Ошибки 404 при доступе к панели администратора**: Убедитесь, что вы используете правильные URL: `/admin/dashboard` и `/admin/help`.
- **Проблемы с аутентификацией в панели**: Проверьте правильность `ADMIN_USERNAME` и `ADMIN_PASSWORD` в вашем `.env` файле (или в переменных окружения системы).
- **Модуль не работает / не отображается**:
  - Убедитесь, что для модуля есть соответствующий файл ключей в папке `configs` (например, `configs/openai_keys.json`) и он содержит хотя бы один валидный ключ.
  - Проверьте статус модуля в панели администратора – он должен быть "Включен".
- **Прокси не используются**:
  - Убедитесь, что переменная окружения `USE_PROXIES` установлена в `true` (или не установлена, тогда используется значение по умолчанию `true`).
  - Проверьте, что в `configs/proxies.json` есть хотя бы один прокси.
  - Проверьте настройку "Использование прокси" в панели администратора.
- **Логи**: При возникновении проблем смотрите в консоль, где запущен Uvicorn – там будут выводиться логи приложения, которые могут помочь диагностировать проблему.
